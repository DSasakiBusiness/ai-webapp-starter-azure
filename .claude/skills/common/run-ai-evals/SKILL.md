# Run AI Evals

## この skill を使う場面
- AI 機能の品質を定量的に評価したい場合
- LLM モデルやプロンプトを変更した後の回帰テスト

## 入力前提
- 評価データセット (入力 → 期待動作のペア) が準備済み
- AiService が動作可能な状態

## 実行手順

### Step 1: 評価データセット準備
```typescript
const evalCases = [
  { input: '挨拶', expected: { hasGreeting: true, isPolite: true } },
  { input: '', expected: { isError: true, hasMessage: true } },
  { input: '非常に長いテキスト...', expected: { isValid: true } },
];
```

### Step 2: 評価基準定義
- **構造評価**: レスポンスに必須フィールドがあるか
- **失敗時動作**: エラー時に適切なメッセージが返るか
- **境界条件**: 空入力、超長文、特殊文字での動作
- **引用**: RAG 使用時に引用元が正しいか
- **再試行**: タイムアウト後のリトライが機能するか

### Step 3: 評価実行
各テストケースを実行し、基準に基づいて Pass/Fail を判定。

### Step 4: レポート生成
```markdown
## AI 評価結果
- 日付: YYYY-MM-DD
- モデル: gpt-4o
- テストケース数: N
- 合格率: X%

| テスト | 結果 | スコア | 詳細 |
|--------|------|--------|------|
| 構造評価 | Pass | 1.0 | ... |
```

## 判断ルール
- 合格率 80% 以上をリリース基準とする
- 構造評価は 100% 必須
- 文字一致ではなく構造・動作で評価する

## 出力形式
評価レポート (Markdown)

## 注意点
- 毎回同じデータセットで評価し、回帰を検出する
- コスト管理のため評価頻度を制限する

## 失敗時の扱い
- 合格率が基準以下: プロンプトの見直し、モデルの変更を ai-engineer に依頼
