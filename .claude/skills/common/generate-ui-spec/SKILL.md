# Generate UI Spec
## この skill を使う場面
- 新しい UI コンポーネントやページの設計時
## 入力前提
- ユーザーストーリーと受け入れ条件が定義済み
## 実行手順
### Step 1: 画面構成の定義
対象ページのワイヤーフレームを Markdown で記述:
- 配置する要素とその順序
- 各要素の役割
- レスポンシブでの変化
### Step 2: インタラクション定義
- 各要素のユーザー操作 (クリック、入力、ホバー)
- 操作後の期待動作
- ローディング状態、エラー状態の表示
### Step 3: データフロー
- API からの取得タイミング
- 状態管理の方針
### Step 4: id/role 属性の定義
E2E テストで使用する id 属性をここで決定。
## 判断ルール
- モバイルファーストで設計
- アクセシビリティ (WCAG 2.1 AA) を考慮
## 出力形式
UI 仕様書 (Markdown)
## 注意点
- デザインツールではなく、Markdown で十分
## 失敗時の扱い
- 仕様が曖昧な場合: product-manager に確認
