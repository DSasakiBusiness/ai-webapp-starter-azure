# Review Release Readiness
## この skill を使う場面
- リリース判定を行う場合
## 入力前提
- 実装が完了し、全テストが通過
## 実行手順
### Step 1: コードチェック
- [ ] lint エラーが 0 件
- [ ] ビルドが成功
- [ ] `any` 型の使用がない
- [ ] `console.log` が残っていない
### Step 2: テストチェック
- [ ] unit テスト通過
- [ ] integration テスト通過
- [ ] E2E テスト通過
- [ ] テストカバレッジ 80% 以上
### Step 3: ドキュメントチェック
- [ ] README 更新済み
- [ ] API ドキュメント更新済み
- [ ] 変更履歴記載済み
### Step 4: セキュリティチェック
- [ ] PentAGI レビュー完了
- [ ] `npm audit` クリア
### Step 5: 判定
全チェック項目が OK → リリース可
1 つでも NG → 該当チームに差し戻し
## 判断ルール
- 全チェック項目の完了が必須
## 出力形式
リリース判定レポート (チェックリスト + 結果)
## 失敗時の扱い
- NG 項目がある場合: 該当 agent に修正を依頼
